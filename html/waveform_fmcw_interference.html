FMCW Radar Mutual Interference: Victim and Interferer Analysis¶

Introduction¶

This notebook demonstrates radar-to-radar interference—a critical challenge in automotive radar where multiple nearby vehicles operate similar radar systems simultaneously. With the proliferation of ADAS (Advanced Driver Assistance Systems) and autonomous vehicles, understanding and mitigating mutual interference is essential for reliable radar operation.

Radar Interference Challenge¶

The Problem:

As automotive radar adoption increases, multiple radars operating in the same frequency band (76-81 GHz) create a congested electromagnetic environment. When radars from different vehicles "see" each other, interference corrupts target detection and tracking.

Interference Scenario:

  • Victim Radar: The radar experiencing interference (our radar system)
  • Interfering Radar: Another radar system causing interference (nearby vehicle)
  • Collision: When both radars transmit/receive simultaneously in overlapping frequency bands

Types of Radar Interference¶

1. In-Band Interference:

  • Same frequency band (76-81 GHz automotive)
  • Direct frequency overlap
  • Most severe impact

2. Mutual FMCW Interference:

  • Both radars using FMCW modulation
  • Chirp collisions create beat frequencies
  • Appears as false targets or noise

3. Temporal Characteristics:

  • Continuous: Interferer always present
  • Intermittent: Sporadic collisions (most common)
  • Synchronized: Worst case (rarely occurs naturally)

Interference Mechanisms¶

FMCW Interference Physics:

When victim radar receives interferer's signal:

  1. Direct Reception: Victim RX picks up interferer TX
  2. Beat Frequency: Mixing creates baseband interference
  3. Range Ambiguity: Interference appears at false ranges
  4. SNR Degradation: Reduces detection capability
  5. False Alarms: Interference mistaken for targets

Interference Power:

$$P_{int} = P_t G_t G_r \left(\frac{\lambda}{4\pi R}\right)^2$$

Where:

  • $P_{int}$ = Received interference power
  • $P_t$ = Interferer transmit power
  • $G_t, G_r$ = Interferer and victim antenna gains
  • $\lambda$ = Wavelength
  • $R$ = Distance between radars

Critical Factor:

Radar-to-radar interference is often much stronger than target returns because interferer directly illuminates victim (no 1/R⁴ loss, only 1/R²).

Interference Patterns¶

Time-Domain Characteristics:

  1. Chirp Overlap: Interference only when chirps collide temporally
  2. Periodic Disruption: Pattern repeats with PRF relationship
  3. Amplitude Spikes: Sudden baseband distortion during collision
  4. Phase Noise: Random phase from unsynchronized interferer

Frequency-Domain Effects:

  1. False Peaks: Appear at beat frequencies
  2. Noise Floor Rise: Reduced sensitivity
  3. Sidelobe Increase: Spectral contamination
  4. Target Masking: Real targets obscured

Interference Parameters¶

Victim Radar (Our System):

  • Down-chirp: 60.6 → 60.4 GHz
  • 4 chirps with frequency diversity (90 MHz steps)
  • PRP: 20 μs, Pulse width: 16 μs
  • Located at origin, facing +X

Interfering Radar (Other Vehicle):

  • Up-chirp: 60.4 → 60.6 GHz (opposite slope)
  • 8 chirps with frequency diversity (70 MHz steps)
  • PRP: 11 μs, Pulse width: 8 μs
  • Located at (30, 0, 0) m, facing -X (towards victim)

Key Differences:

  • Opposite chirp direction: Down vs. up
  • Different PRF: 50 kHz vs. 91 kHz
  • Different pulse count: 4 vs. 8
  • Frequency diversity: Both use frequency hopping (mitigation attempt)

Mitigation Strategies¶

Time Domain:

  1. Random PRP: Avoid synchronization
  2. Short Pulses: Reduce collision probability
  3. Burst Timing: Coordinate transmission times

Frequency Domain:

  1. Frequency Hopping: Change carrier per chirp (used in this example)
  2. Wide Bandwidth: Spread interference energy
  3. Band Segmentation: Divide 76-81 GHz into channels

Spatial Domain:

  1. Beam Steering: Point away from interferer
  2. Polarization Diversity: Orthogonal polarizations
  3. Antenna Pattern: Null steering

Signal Processing:

  1. Interference Detection: Identify corrupted samples
  2. Blanking: Zero out interference periods
  3. Subtraction: Estimate and remove interference
  4. Robust Algorithms: CFAR with interference rejection
  5. Deep Learning: Neural networks for classification

Automotive Radar Context¶

Typical Scenarios:

  1. Highway: Multiple vehicles with forward-facing radar
  2. Intersection: Radars from different directions
  3. Parking: Dense radar environment
  4. Convoy: Following vehicles with similar systems

Frequency Allocation:

  • 76-77 GHz: Long-range radar (LRR)
  • 77-81 GHz: Short/medium-range radar (SRR/MRR)
  • Shared Band: Interference inevitable

Industry Response:

  • IEEE, ETSI standards for coexistence
  • Interference mitigation mandatory for certification
  • Ongoing research in robust waveforms

# Import necessary modules for radar simulation
import numpy as np
from radarsimpy import Radar, Transmitter, Receiver

### Interfering Radar Configuration ###

# Frequency offsets for frequency diversity (8 chirps)
# Each chirp hops to different frequency band (70 MHz steps)
f_offset_int = np.arange(0, 8) * 70e6  # [0, 70, 140, ..., 490] MHz

# Configure interfering radar transmitter
int_tx = Transmitter(
    f=[60.4e9, 60.6e9],     # Frequency sweep: 60.4-60.6 GHz (200 MHz BW, up-chirp)
    t=[0, 8e-6],            # Chirp duration: 0-8 μs
    tx_power=15,            # Transmit power: 15 dBm (~32 mW)
    prp=11e-6,              # Pulse repetition period: 11 μs (90.9 kHz PRF)
    pulses=8,               # Number of chirps: 8
    f_offset=f_offset_int,  # Frequency diversity: different carrier per chirp
    channels=[
        dict(
            location=(0, 0.1, 0),  # Antenna position: 0.1m offset in Y
            pulse_phs=np.array([0, 0, 180, 0, 0, 0, 0, 0])  # Phase coding per chirp
        )
    ],
)

# Configure interfering radar receiver
int_rx = Receiver(
    fs=20e6,             # Sampling rate: 20 MHz
    noise_figure=8,      # Noise figure: 8 dB
    rf_gain=20,          # RF gain: 20 dB
    load_resistor=500,   # Load resistance: 500 Ω
    baseband_gain=30,    # Baseband gain: 30 dB
    channels=[dict(location=(0, 0.1, 0))],  # Receiver antenna
)

# Create interfering radar system
# Location: (30, 0, 0) m → 30m from victim
# Rotation: (180, 0, 0) deg → Facing -X direction (towards victim)
int_radar = Radar(
    transmitter=int_tx, 
    receiver=int_rx, 
    location=(30, 0, 0),    # Position: 30m along X-axis
    rotation=(180, 0, 0)    # Orientation: 180° yaw (facing victim)
)

Victim Radar Parameters¶

Key Differences from Interferer:

  • Chirp Direction: Down-chirp (60.6 → 60.4 GHz) vs. up-chirp
  • Pulse Width: 16 μs vs. 8 μs (longer)
  • PRF: 50 kHz vs. 90.9 kHz (slower)
  • Pulse Count: 4 vs. 8 (fewer chirps)
  • Frequency Hop: 90 MHz steps vs. 70 MHz steps
  • Power: 25 dBm vs. 15 dBm (higher)

Interference Integration:

The interf parameter in the simulation links the interfering radar to the victim radar, enabling realistic mutual interference modeling.

Frequency Diversity:

Both radars use frequency hopping to reduce interference probability. However, with overlapping bands (60.4-60.6 GHz), collisions are inevitable when chirps overlap temporally.

### Victim Radar Configuration ###

# Frequency offsets for frequency diversity (4 chirps)
# Each chirp hops to different frequency band (90 MHz steps)
f_offset_vit = np.arange(0, 4) * 90e6  # [0, 90, 180, 270] MHz

# Configure victim radar transmitter
tx = Transmitter(
    f=[60.6e9, 60.4e9],     # Frequency sweep: 60.6-60.4 GHz (200 MHz BW, down-chirp)
    t=[0, 16e-6],           # Chirp duration: 0-16 μs (longer than interferer)
    tx_power=25,            # Transmit power: 25 dBm (~316 mW, higher than interferer)
    prp=20e-6,              # Pulse repetition period: 20 μs (50 kHz PRF)
    pulses=4,               # Number of chirps: 4 (fewer than interferer)
    f_offset=f_offset_vit,  # Frequency diversity: different carrier per chirp
    channels=[
        dict(
            location=(0, 0, 0),  # Antenna position: at origin
            pulse_phs=np.array([180, 0, 0, 0])  # Phase coding: first chirp 180°
        )
    ],
)

# Configure victim radar receiver
rx = Receiver(
    fs=40e6,             # Sampling rate: 40 MHz (higher than interferer)
    noise_figure=2,      # Noise figure: 2 dB (lower noise, better sensitivity)
    rf_gain=20,          # RF gain: 20 dB
    load_resistor=500,   # Load resistance: 500 Ω
    baseband_gain=60,    # Baseband gain: 60 dB (higher than interferer)
    channels=[dict(location=(0, 0, 0))],  # Receiver antenna at origin
)

# Create victim radar system
# Location: (0, 0, 0) m → At origin
# Rotation: (0, 0, 0) deg → Facing +X direction (default)
radar = Radar(transmitter=tx, receiver=rx)

Radar Interference Simulation¶

Simulate victim radar operation with interference from the interfering radar.

Simulation Configuration¶

Interference Modeling:

The interf parameter in sim_radar enables mutual interference simulation:

  • Computes when victim RX receives interferer TX
  • Calculates interference power (1/R² propagation)
  • Generates beat frequencies from chirp mixing
  • Adds interference to victim baseband

Output Structure:

The simulation returns three separate components:

  1. baseband: Clean target returns (no interference)
  2. interference: Pure interference signal
  3. Total: baseband + interference (realistic scenario)

This separation enables:

  • Analysis of interference-free performance
  • Quantification of interference impact
  • Development of mitigation algorithms
  • Before/after comparison

Data Dimensions:

  • Channels: 1 (single victim TX/RX)
  • Pulses: 4 (victim radar chirps)
  • Samples: 640 per chirp (40 MHz × 16 μs)

Computational Note:

Interference simulation adds computational complexity as it models electromagnetic propagation between radars in addition to target returns.

In [5]:
# Import radar simulator and timing module
from radarsimpy.simulator import sim_radar
import time

# Start timing
tic = time.time()

# Simulate victim radar with interference from interfering radar
# interf=int_radar: Links interfering radar to victim for mutual interference
bb_data = sim_radar(radar, targets, interf=int_radar)

# Extract simulation outputs
timestamp = bb_data["timestamp"]        # Time axis for each sample [1, 4, 640]
baseband = bb_data["baseband"]          # Clean target returns (no interference)
interf = bb_data["interference"]        # Pure interference component

# Combine clean and interference for realistic scenario
interf_bb = baseband + interf           # Total baseband with interference

# End timing
toc = time.time()
print("Exec time:", toc - tic, "s")

Analysis: Interference Correlation¶

Key Observations:

From the visualization, interference corruption is clearly visible in the baseband when victim and interferer chirps overlap temporally and spectrally:

Temporal Overlap:

  • Interferer has shorter PRP (11 μs) and transmits more frequently (8 chirps vs. 4)
  • Multiple interferer chirps can collide with a single victim chirp
  • Clean periods occur when no temporal overlap exists

Spectral Overlap:

  • Both radars operate in 60.4-60.6 GHz band (200 MHz)
  • Frequency hopping reduces collision probability but doesn't eliminate it
  • When hopped frequencies overlap during temporal collision → strong interference

Baseband Corruption Pattern:

  • Amplitude spikes: Sudden increases during interference
  • Phase distortion: Random phase from unsynchronized interferer
  • Intermittent nature: Not continuous, depends on chirp timing
  • Unpredictable timing: Asynchronous PRFs create varying collision patterns

Mitigation Implications:

  1. Detection: Amplitude threshold can identify corrupted samples
  2. Blanking: Zero out or interpolate through corruption
  3. Frequency diversity: Partially helps but incomplete protection
  4. Statistical methods: CFAR with interference rejection
  5. Machine learning: Classify and remove interference patterns

This simulation demonstrates the realistic challenge of automotive radar coexistence and the need for robust interference mitigation in signal processing.
