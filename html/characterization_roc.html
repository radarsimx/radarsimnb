Receiver Operating Characteristic (ROC) Analysis for Radar Systems¶

Introduction¶

This notebook demonstrates how to use RadarSimPy's ROC analysis tools to characterize radar detection performance. You'll learn how to:

  • Understand the fundamental concepts of radar detection theory
  • Calculate probability of detection $(P_d)$ for given SNR and false alarm rate $(P_{fa})$
  • Determine required SNR for target detection specifications
  • Compare different target fluctuation models (Swerling cases)
  • Analyze the impact of non-coherent integration on detection performance
  • Use RadarSimPy's ROC analysis tools for system design

What is ROC Analysis?

Receiver Operating Characteristic (ROC) analysis is essential for radar system design and performance prediction. It quantifies the trade-off between:

  • Detection probability $(P_d)$: Likelihood of correctly detecting a target when present
  • False alarm probability $(P_{fa})$: Likelihood of incorrectly declaring a target when absent

ROC curves help radar engineers:

  • Set detection thresholds for specific operational requirements
  • Predict system performance under different conditions
  • Compare different radar configurations
  • Design systems to meet detection specifications

Key Concepts¶

Detection Theory Fundamentals:

In radar detection, we test two hypotheses:

  • $H_0$ (Null hypothesis): No target present (noise only)
  • $H_1$ (Alternative hypothesis): Target present (signal + noise)

Performance Metrics:

  1. Probability of Detection $(P_d)$:

    • Probability that the detector correctly identifies a target
    • Also called "sensitivity" or "true positive rate"
    • Higher is better (ideally close to 1)

  2. Probability of False Alarm $(P_{fa})$:

    • Probability that the detector incorrectly declares a target
    • Also called "false positive rate"
    • Lower is better (typically $10^{-4}$ to $10^{-6}$)

  3. Signal-to-Noise Ratio (SNR):

    • Ratio of signal power to noise power
    • Fundamental parameter determining detection performance
    • Measured in dB: $SNR_{dB} = 10\log_{10}(SNR_{linear})$

The ROC Curve:

An ROC curve plots $P_d$ versus $P_{fa}$ for various detection thresholds. For radar, we often fix $P_{fa}$ and examine how $P_d$ varies with SNR.

Swerling Target Models:

Real targets don't have constant RCS—they fluctuate due to:

  • Aspect angle changes
  • Target rotation
  • Multipath effects
  • Component scattering interference

The Swerling models characterize RCS fluctuation statistics:

Model Description Fluctuation Rate Distribution
Swerling 1 Complex target, slow fluctuation Scan-to-scan Exponential (Rayleigh amplitude)
Swerling 2 Complex target, fast fluctuation Pulse-to-pulse Exponential (Rayleigh amplitude)
Swerling 3 Single dominant scatterer, slow Scan-to-scan Chi-squared with 4 DOF
Swerling 4 Single dominant scatterer, fast Pulse-to-pulse Chi-squared with 4 DOF
Swerling 5 (or 0) (Non-fluctuating) Constant RCS No fluctuation Deterministic
Coherent Ideal coherent integration N/A Assumes perfect coherence
Real Real signal with noise N/A Based on actual distributions

Integration Gain:

Combining multiple observations improves detection:

  • Coherent integration: Adds signals in-phase (requires phase coherence)
    • Gain = $N$ (linear) or $10\log_{10}(N)$ dB
  • Non-coherent integration: Adds signal envelopes or powers
    • Gain < $N$ due to loss of phase information
    • Used when phase coherence cannot be maintained

Practical Example:

For an automotive radar detecting a car at 100m:

  • Required $P_d = 0.9$ (90% detection probability)
  • Acceptable $P_{fa} = 10^{-5}$ (one false alarm per 100,000 samples)
  • Target type: Swerling 1 (complex vehicle, scan-to-scan fluctuation)
  • Integration: N = 10 pulses

This notebook shows how to calculate the required SNR for these specifications.

Calculate $P_d$ from SNR and $P_{fa}$¶

Given SNR and false alarm probability, we can predict detection performance.

Example 1: Swerling 3 Targets (Moderate Fluctuation)¶

Scenario:

  • Target type: Swerling 3 (single dominant scatterer, slow fluctuation)
  • Integration samples: N = 64
  • False alarm rates: $P_{fa} = 10^{-4}$ and $10^{-5}$
  • SNR range: -10 to 10 dB

This represents a target with one strong reflector and several weak ones, with RCS changing scan-to-scan.

Calculate Detection Probability¶

Use roc_pd() to calculate probability of detection for:

  • Multiple false alarm rates (array of $P_{fa}$ values)
  • Range of SNR values (-10 to 10 dB)
  • N = 64 non-coherent integration samples
  • Swerling 3 target model

from radarsimpy.tools import roc_pd, roc_snr
import numpy as np

snr = np.arange(-10, 10, 0.1)
pfa = np.array([1e-4, 1e-5])
pd = roc_pd(pfa, snr, 64, "Swerling 3")

Example 2: Effect of Integration (N = 20 vs N = 64)¶

Now let's reduce the integration samples from 64 to 20 and observe the impact on detection performance.

Expected behavior:

  • Fewer integration samples → less integration gain
  • Lower SNR improvement → detection curves shift right
  • Need higher SNR to achieve same $P_d$

This demonstrates the integration gain trade-off: More samples improve detection but increase processing time and limit maximum unambiguous Doppler.

Now we flip the problem: Given detection requirements, what SNR do we need?

This is the inverse calculation—critical for radar system design. Instead of predicting $P_d$ from SNR, we determine the minimum SNR needed to meet detection specifications.

Design Guidelines¶

Choosing False Alarm Rate:

  • $P_{fa} = 10^{-3}$: High false alarm rate, not typically used
  • $P_{fa} = 10^{-4}$ to $10^{-5}$: Common for tracking radars
  • $P_{fa} = 10^{-6}$ to $10^{-8}$: Low false alarms for critical applications

Choosing Detection Probability:

  • $P_d = 0.5$: Marginal detection, not reliable
  • $P_d = 0.9$: Standard requirement for most applications
  • $P_d = 0.99$: High reliability for safety-critical systems

Target Model Selection:

  • Swerling 1: Worst case (ships, aircraft with deep fades)
  • Swerling 2: Complex targets with fast fluctuation
  • Swerling 3/4: Most realistic for automotive/aerospace
  • Swerling 5: Best case (simple, constant RCS targets)

Integration Strategy:

  • More samples → Better detection, longer processing time
  • Typical range: N = 8 to 64 for most applications
  • Balance detection gain with Doppler ambiguity
  • Consider computational complexity
