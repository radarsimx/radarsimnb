FMCW Automotive Radar: Vehicle Detection and Tracking¶

Introduction¶

This notebook demonstrates a complete FMCW (Frequency-Modulated Continuous Wave) automotive radar simulation, including vehicle detection, range measurement, and velocity estimation. FMCW radar is the dominant technology in modern automotive safety systems (ADAS) and autonomous vehicles.

FMCW Radar Fundamentals¶

FMCW radar transmits a continuous signal whose frequency varies linearly over time (chirp). By comparing the transmitted and received signals, both range and velocity can be simultaneously measured.

Operating Principle:

  1. Frequency Modulation: Transmit a linear chirp from f_start to f_end
  2. Echo Reception: Receive reflections from targets at different ranges
  3. Mixing: Multiply received signal with transmitted signal (dechirp)
  4. Beat Frequency: Resulting IF signal has frequency proportional to range

Key Parameters:

$$f_{beat} = \frac{2 \cdot B \cdot R}{c \cdot T_c}$$

Where:

  • $f_{beat}$ = Beat frequency (Hz)
  • $B$ = Chirp bandwidth (Hz)
  • $R$ = Target range (m)
  • $c$ = Speed of light (3×10⁸ m/s)
  • $T_c$ = Chirp duration (s)

Range and Velocity Measurement¶

Range Calculation:

$$R = \frac{f_{beat} \cdot c \cdot T_c}{2 \cdot B}$$

Velocity via Doppler:

Multiple chirps are transmitted to measure Doppler shift:

$$v = \frac{f_d \cdot \lambda}{2} = \frac{f_d \cdot c}{2 \cdot f_c}$$

Where:

  • $v$ = Radial velocity (m/s)
  • $f_d$ = Doppler frequency (Hz)
  • $\lambda$ = Wavelength (m)
  • $f_c$ = Carrier frequency (Hz)

FMCW Radar Advantages¶

1. Simultaneous Range and Velocity

  • Single waveform measures both parameters
  • No range-Doppler ambiguity trade-off
  • Efficient for tracking multiple targets

2. High Range Resolution

  • Resolution: ΔR = c/(2B)
  • Wide bandwidth (100-500 MHz) → cm-level accuracy
  • Independent of pulse duration

3. Continuous Operation

  • No blind ranges during pulse transmission
  • Close-range detection capability
  • Ideal for short-range automotive applications

4. Low Peak Power

  • Continuous transmission spreads energy over time
  • Smaller, lighter, cheaper components
  • Better EMC characteristics

5. Excellent SNR

  • Long integration time improves sensitivity
  • Coherent processing gain from multiple chirps
  • Good performance in adverse weather

Automotive Radar Applications¶

Advanced Driver Assistance Systems (ADAS):

  • Adaptive Cruise Control (ACC): Maintain safe following distance
  • Collision Warning: Alert driver of imminent crash
  • Automatic Emergency Braking (AEB): Apply brakes automatically
  • Blind Spot Detection: Monitor adjacent lanes
  • Lane Change Assist: Detect vehicles in blind spots
  • Cross Traffic Alert: Detect crossing vehicles when backing up

Autonomous Driving:

  • 360° Perception: Multiple radars for complete coverage
  • Sensor Fusion: Combine with cameras and lidar
  • All-Weather Operation: Reliable in rain, fog, snow
  • Long-Range Detection: 200+ meters for highway driving

Signal Processing Pipeline¶

Two-Dimensional FFT Processing:

  1. Range FFT:

    • FFT across fast-time (ADC samples)
    • Converts beat frequency to range
    • Output: Range profiles for each chirp

  2. Doppler FFT:

    • FFT across slow-time (chirps/pulses)
    • Converts phase change to velocity
    • Output: Range-Doppler map

  3. Detection:

    • CFAR (Constant False Alarm Rate) thresholding
    • Peak detection in range-Doppler space
    • Extract target range and velocity

Define FMCW Radar Transmitter¶

Configure the chirp waveform and transmission parameters.

Chirp Waveform Definition:

The FMCW chirp is defined by frequency and time arrays:

  • f = [f_start, f_end]: Frequency sweeps from 76.95 GHz to 77.05 GHz
  • t = [t_start, t_end]: Chirp duration (if single value, assumes start at 0)

Transmitter Parameters:

  • Frequency Range: 76.95-77.05 GHz

    • Center frequency: 77 GHz (automotive W-band)
    • Bandwidth: B = 100 MHz
    • Range resolution: ΔR = c/(2B) = 1.5 m

  • Chirp Duration: 80 μs

    • Fast-time sample period
    • Determines maximum unambiguous range

  • Transmit Power: 40 dBm (10 W)

    • Typical for long-range automotive radar
    • Ensures adequate SNR at 200+ meters

  • Pulse Repetition Period (PRP): 100 μs

    • Time between chirp starts
    • Determines maximum unambiguous velocity
    • PRF = 1/PRP = 10 kHz

  • Number of Pulses: 512

    • Chirps in coherent processing interval (CPI)
    • More pulses → better Doppler resolution
    • Doppler resolution: Δv = λ/(2×512×PRP) ≈ 0.038 m/s

Range and Velocity Limits:

  • Maximum Range: R_max = c×fs×T_c/(2×B) ≈ 240 m
  • Unambiguous Velocity: v_max = λ/(2×PRP) = 19.5 m/s (±70 km/h)
In [4]:
# Configure FMCW transmitter
tx = Transmitter(
    f=[77e9 - 50e6, 77e9 + 50e6],  # Frequency sweep: 76.95-77.05 GHz (100 MHz BW)
    t=80e-6,                        # Chirp duration: 80 μs
    tx_power=40,                    # Transmit power: 40 dBm (10 W)
    prp=100e-6,                     # Pulse repetition period: 100 μs (10 kHz PRF)
    pulses=512,                     # Number of chirps in CPI: 512
    channels=[tx_channel],          # Transmitter antenna configuration
)

Define Radar Receiver¶

Configure receiver parameters for optimal signal processing.

Receiver Parameters:

  • Sampling Rate (fs): 2 MHz

    • Nyquist requirement for beat frequencies
    • Determines maximum unambiguous range
    • R_max = c×fs×T_c/(2×B) = 3×10⁸ × 2×10⁶ × 80×10⁻⁶/(2×100×10⁶) = 240 m

  • Noise Figure: 8 dB

    • Receiver noise performance
    • Typical for automotive radar front-ends
    • Affects maximum detection range

  • RF Gain: 20 dB

    • Low-noise amplifier (LNA) gain
    • Amplifies weak echo signals
    • Before mixer stage

  • Load Resistor: 500 Ω

    • Converts RF power to voltage
    • Impedance matching component

  • Baseband Gain: 30 dB

    • Amplification after mixing (IF stage)
    • Prepares signal for ADC
    • Total gain: 20 + 30 = 50 dB

Signal Chain: Antenna → LNA (20 dB) → Mixer → IF Amplifier (30 dB) → ADC (2 MHz)

In [6]:
# Configure radar receiver

    baseband_gain=30,    # Baseband/IF gain: 30 dB

Ground Plane Effects:

The ground plane with εᵣ = 3.2 + 0.1j models:

  • Real part (3.2): Dielectric constant of asphalt
  • Imaginary part (0.1): Losses/absorption in material
  • Multipath: Reflections from ground create interference
  • Ground bounce: Increases effective target RCS at low angles

Radar Scene Simulation¶

Simulate FMCW radar returns from the vehicle and ground plane using ray tracing.

Simulation Parameters¶

Ray Tracing Configuration:

  • Density: 0.1 rays per wavelength²

    • Balance between accuracy and computation time
    • Sufficient for automotive-sized targets
    • Lower density acceptable for large, flat surfaces

  • Debug: False

    • Suppresses detailed progress output
    • Set to True for troubleshooting

Output Data Structure:

The simulation returns a dictionary containing:

  • baseband: Complex I/Q samples [channels, pulses, samples]
  • noise: System noise contribution [channels, pulses, samples]
  • timestamp: Time stamps for each sample

Dimensions:

  • Channels: 1 (single Tx/Rx pair)
  • Pulses: 512 (chirps in CPI)
  • Samples: ~160 per chirp (2 MHz × 80 μs)

Expected Computation:

  • Typical runtime: 10-60 seconds
  • Depends on target complexity and ray density
  • Progress displayed during execution

Radar Signal Processing¶

Apply two-dimensional FFT processing to extract range and velocity information.

Range FFT Processing¶

Range FFT (also called dechirp FFT) converts beat frequencies to range bins.

Processing Steps:

  1. Windowing: Apply Chebyshev window to reduce sidelobes

    • 60 dB sidelobe suppression
    • Reduces spectral leakage
    • Improves dynamic range for weak targets near strong ones

  2. FFT: Transform across fast-time (ADC samples within chirp)

    • Input: Baseband samples [channels, pulses, samples]
    • Output: Range profiles [channels, pulses, range_bins]

  3. Range Resolution: ΔR = c/(2×B) = 3×10⁸/(2×100×10⁶) = 1.5 m

Window Function Choice:

Chebyshev window provides:

  • Excellent sidelobe suppression (60 dB)
  • Good main lobe width
  • Optimal for automotive scenarios with strong ground returns
In [11]:
# Import signal processing modules
from scipy import signal
import radarsimpy.processing as proc

# Create Chebyshev window for range FFT (60 dB sidelobe suppression)
range_window = signal.windows.chebwin(radar.sample_prop["samples_per_pulse"], at=60)

# Perform range FFT to convert beat frequencies to range
# Input: baseband [channels, pulses, samples]
# Output: range_profile [channels, pulses, range_bins]
range_profile = proc.range_fft(baseband, range_window)

Doppler FFT Processing¶

Doppler FFT analyzes phase changes across chirps to extract velocity information.

Processing Steps:

  1. Windowing: Apply Chebyshev window across slow-time (pulses)

    • 60 dB sidelobe suppression
    • Reduces Doppler sidelobes
    • Important for detecting weak targets near strong ones

  2. FFT: Transform across pulses

    • Input: Range profiles [channels, pulses, range_bins]
    • Output: Range-Doppler map [channels, Doppler_bins, range_bins]

  3. Velocity Resolution:

    • Δv = λ/(2×N×PRP)
    • Δv = 0.0039/(2×512×100×10⁻⁶) ≈ 0.038 m/s

Doppler Interpretation:

  • Negative Doppler: Target approaching (closing velocity)
  • Zero Doppler: Stationary targets (ground clutter)
  • Positive Doppler: Target receding (opening velocity)

For our scenario:

  • Vehicle at -10 m/s → Doppler frequency ≈ -5.1 kHz
In [12]:
# Create Chebyshev window for Doppler FFT (60 dB sidelobe suppression)
doppler_window = signal.windows.chebwin(
    radar.radar_prop["transmitter"].waveform_prop["pulses"], at=60
)

# Perform Doppler FFT to extract velocity information
# Input: range_profile [channels, pulses, range_bins]
# Output: range_doppler [channels, Doppler_bins, range_bins]
range_doppler = proc.doppler_fft(range_profile, doppler_window)

Visualize Range-Doppler Map¶

Generate a 3D surface plot showing target detection in range-velocity space.

Axis Calculations:

  1. Range Axis:

    • Maximum range: R_max = c×fs×T_c/(2×B)
    • Linearly spaced from 0 to R_max

  2. Doppler/Velocity Axis:

    • Unambiguous velocity: v_max = λ/(2×PRP) = c/(2×f_c×PRP)
    • Range: -v_max to 0 (negative for closing targets)
    • Note: Sign convention may vary by implementation

Expected Features:

  1. Vehicle Peak: Strong return at ~150m, -10 m/s
  2. Ground Clutter: Strong returns at zero Doppler across all ranges
  3. Multipath: Secondary peaks from ground bounce
  4. Noise Floor: Background level showing system noise

Interpretation:

  • Peak Location: (Range, Velocity) → (150 m, -10 m/s)
  • Peak Amplitude: Indicates target RCS and SNR
  • Width: Determined by resolution and windowing
In [13]:
# Calculate maximum unambiguous range
max_range = (
    3e8  # Speed of light
    * radar.radar_prop["receiver"].bb_prop["fs"]  # Sampling rate
    * radar.radar_prop["transmitter"].waveform_prop["pulse_length"]  # Chirp duration
    / radar.radar_prop["transmitter"].waveform_prop["bandwidth"]  # Chirp bandwidth
    / 2  # Two-way propagation
)

# Calculate unambiguous velocity (maximum Doppler)
unambiguous_speed = (
    3e8  # Speed of light
    / radar.radar_prop["transmitter"].waveform_prop["prp"][0]  # Pulse repetition period
    / 77e9  # Carrier frequency
    / 2  # Two-way Doppler
)

# Create range axis (0 to max_range)
range_axis = np.linspace(
    0, max_range, radar.sample_prop["samples_per_pulse"], endpoint=False
)

# Create Doppler/velocity axis (-v_max to 0)
# Negative velocities indicate closing (approaching) targets
doppler_axis = np.linspace(
    -unambiguous_speed,
    0,
    radar.radar_prop["transmitter"].waveform_prop["pulses"],
    endpoint=False,
)

# Create 3D surface plot of range-Doppler map
fig = go.Figure()
fig.add_trace(
    go.Surface(
        x=range_axis,                                    # Range axis (m)
        y=doppler_axis,                                  # Velocity axis (m/s)
