OTFS Radar: Orthogonal Time Frequency Space¶

OTFS (Orthogonal Time Frequency Space) modulates data symbols in the delay-Doppler (DD) domain instead of the time-frequency domain used by OFDM. Every DD-domain symbol is spread — via a 2D transform — across all time slots and all subcarriers of the transmitted frame, so each symbol experiences the full time-varying, frequency-selective channel. This gives OTFS strong robustness to Doppler spread in high-mobility channels, and — for radar sensing — lets range and velocity be read out jointly from a single frame.

Delay-Doppler data grid $x[k,l]$: $k = 0, \dots, N-1$ indexes Doppler bins, $l = 0, \dots, M-1$ indexes delay bins.

ISFFT (Inverse Symplectic Finite Fourier Transform) maps the DD grid to the time-frequency (TF) grid $X[n,m]$ ($n$ = OTFS symbol/time-slot index, $m$ = subcarrier index):

$$X[n,m] = \sum_{k=0}^{N-1}\sum_{l=0}^{M-1} x[k,l]\, e^{\,j2\pi\left(\frac{nk}{N} - \frac{ml}{M}\right)} \;=\; \mathrm{IFFT}_{k\to n}\Big\{\mathrm{FFT}_{l\to m}\{x\}\Big\}$$

Heisenberg transform then turns each TF row $X[n,:]$ into a time-domain block via an IFFT — exactly like one OFDM symbol — and the $N$ blocks are transmitted back-to-back to form the full OTFS frame:

$$s[n,q] = \mathrm{IFFT}_m\{X[n,:]\}[q], \qquad q = 0,\dots,M-1$$

Embedded pilot for sensing: a single delay-Doppler bin $x[0,0]$ carries a strong pilot impulse, surrounded by an all-zero guard region large enough to cover the expected delay/Doppler spread of all targets; the rest of the grid carries ordinary QPSK payload data. This is the standard OTFS channel-estimation/radar-sensing technique — it avoids ever dividing by the (non-constant-modulus) transmitted data, which would otherwise blow up the noise floor.

At the receiver, the reverse steps recover range and velocity jointly, with no data removal needed:

  1. Wigner transform — per-block FFT converts each received time-domain block back to the TF grid: $Y[n,m] = \mathrm{FFT}_q\{y[n,:]\}$
  2. SFFT (inverse of ISFFT) — FFT over $n$ then IFFT over $m$ converts $Y[n,m]$ back into the delay-Doppler domain $y[k,l]$

Because the guard region around the pilot is otherwise empty, every target's reflection shows up as a clean, scaled/shifted copy of the pilot at its own $(\Delta k, \Delta l)$ position in $y[k,l]$ — directly readable as range and velocity, with no matched filtering or division required:

$$\Delta R = \frac{c}{2 M \Delta f}, \qquad v_{max} = \frac{c}{2 f_c \, T_{sym}}$$

FMCW vs. OFDM vs. OTFS¶

Feature FMCW OFDM OTFS
Modulation Frequency chirp Multi-carrier phase/amplitude (single TF symbol, repeated) Multi-carrier phase/amplitude, spread over a full 2D delay-Doppler frame
Data domain None Time-frequency (per symbol) Delay-Doppler (per frame)
Range Beat frequency Subcarrier phase ramp (per-symbol IFFT) Joint 2D transform (SFFT) of the whole frame
Velocity Slow-time FFT Slow-time FFT across identical symbols Same SFFT step as range — extracted jointly, not as a separate stage
Doppler robustness Sensitive to intra-chirp Doppler Sensitive to inter-carrier interference from Doppler Designed to spread each symbol's energy across the whole time-frequency plane
Data payload None Full — inherently a comms waveform Full outside the pilot's guard region — inherently a comms waveform

Generate the Delay-Doppler Data Grid, then ISFFT + Heisenberg Transform¶

  1. Fill the $N \times M$ delay-Doppler grid with random QPSK payload data
  2. Reserve a small all-zero guard region around bin $(0,0)$ — large enough to cover every target's delay/Doppler shift — and place a strong pilot impulse at $(0,0)$ for sensing
  3. ISFFT: FFT along the delay axis, then IFFT along the Doppler axis → time-frequency grid tf_data[n, m]
  4. Heisenberg transform: IFFT along the subcarrier axis (per row) → otfs_time[n, q], one time-domain block per OTFS symbol
  5. Concatenate all $N$ blocks into a single time-domain frame and normalize to unit average power

Unlike the OFDM example (which repeats the same symbol every pulse), every block here is different — this is exactly OTFS's defining property, and why the whole frame must be built as a single continuous waveform.

In [30]:
# Generate random QPSK payload data on every delay-Doppler grid bin (reproducible)
np.random.seed(1)
bits_i = np.random.randint(0, 2, (N, M))
bits_q = np.random.randint(0, 2, (N, M))
dd_data = (2 * bits_i - 1 + 1j * (2 * bits_q - 1)) / np.sqrt(2)  # Unit-power QPSK, shape (N, M)

# Embedded pilot + guard region (standard OTFS radar/channel-estimation technique):
# reserve a small all-zero guard box around a single pilot impulse at (0,0), sized to
# cover every target's delay bin (0..~110) and Doppler bin (+/-10), then zero out any
# payload data that would otherwise fall inside it.
guard_delay = 110  # Delay guard: bins [0, guard_delay)
guard_doppler = 10  # Doppler guard: bins [-guard_doppler, guard_doppler] (wrapped)
doppler_idx = np.arange(N)
doppler_guard_mask = (doppler_idx <= guard_doppler) | (doppler_idx >= N - guard_doppler)
dd_data[np.ix_(doppler_guard_mask, np.arange(guard_delay))] = 0
dd_data[0, 0] = np.sqrt(N * M)  # Pilot impulse, boosted so it stands out after normalization


def isfft(x):
    """Inverse Symplectic Finite Fourier Transform: delay-Doppler -> time-frequency."""
    return np.fft.ifft(np.fft.fft(x, axis=1), axis=0)


# Time-frequency domain grid actually transmitted
tf_data = isfft(dd_data)  # shape (N, M): axis 0 = OTFS symbol n, axis 1 = subcarrier m

# Heisenberg transform: per-block IFFT over subcarriers -> time-domain blocks
otfs_time = np.fft.ifft(tf_data, axis=1)  # shape (N, M)

# Concatenate the N blocks into one continuous frame and normalize to unit average power
otfs_frame = otfs_time.reshape(-1)
otfs_frame = otfs_frame / np.sqrt(np.mean(np.abs(otfs_frame) ** 2))

# Waveform modulation arrays for the transmitter channel (single continuous pulse)
mod_t = np.arange(N * M) * Ts  # Time stamps spanning the full frame
amp = np.abs(otfs_frame)  # Relative amplitude (envelope)
phs = np.angle(otfs_frame, deg=True)  # Phase in degrees

OTFS Radar Signal Processing¶

Wigner Transform + SFFT: Joint Delay-Doppler Map¶

Because sensing relies on the embedded pilot rather than dividing out the transmitted data, range and velocity are recovered directly, in one joint 2D transform:

  1. Wigner transform (inverse of the Heisenberg transform) — per-block FFT converts the time-domain samples back to the time-frequency grid: $Y[n,m] = \mathrm{FFT}_m\{y[n,:]\}$
  2. A Chebyshev window (60 dB sidelobe suppression) is applied across blocks ($n$, the Doppler axis) to control sidelobe leakage between targets
  3. SFFT (inverse of the ISFFT) — FFT over $n$ (blocks → Doppler bins), then IFFT over $m$ (subcarriers → delay bins) — converts $Y[n,m]$ directly into the delay-Doppler grid $y[k,l]$

Every target's reflection appears in $y[k,l]$ as a scaled, shifted copy of the pilot at its own $(\Delta k, \Delta l)$ — no data removal or division step is needed, unlike the OFDM example.

Result: a delay-Doppler map [256 Doppler bins × 256 range bins], with the same 1.5 m range resolution as the OFDM example.

In [39]:
# Step 1: Wigner transform - per-block FFT back to the time-frequency (subcarrier) grid
rx_tf = np.fft.fft(rx_frame, axis=1)  # shape (N, M)

# Step 2: Chebyshev window across blocks (Doppler axis) to control sidelobe leakage
doppler_window = signal.windows.chebwin(N, at=60)
rx_tf_windowed = rx_tf * doppler_window[:, np.newaxis]

# Step 3: SFFT - FFT over blocks (Doppler), then IFFT over subcarriers (delay) -> delay-Doppler grid
rx_dd = np.fft.ifft(np.fft.fft(rx_tf_windowed, axis=0), axis=1)  # shape (N, M): axis 0 = k, axis 1 = l

# Center zero Doppler
range_doppler = np.fft.fftshift(rx_dd, axes=0)

# Maximum unambiguous velocity: v_max = c / (2 * fc * T_sym)
unambiguous_speed = 3e8 / T_sym / fc / 2

Summary¶

This notebook demonstrated a complete OTFS radar simulation and delay-Doppler processing chain:

  • Data is placed on the delay-Doppler grid: a single pilot impulse (with a guard region) for radar sensing, plus random QPSK payload elsewhere — then converted to the time domain via ISFFT (delay-Doppler → time-frequency) followed by a per-block IFFT (Heisenberg transform, the same math as one OFDM symbol). Unlike OFDM, every block carries different content, so the whole 256-block frame is transmitted as a single continuous pulse.
  • The receiver reverses this with a per-block FFT (Wigner transform) and the SFFT (an FFT over Doppler and an IFFT over delay) to read range and velocity directly off the resulting delay-Doppler map — no division or data-removal step is needed, because the guard region keeps the pilot's neighborhood free of payload interference. (Dividing by the transmitted data directly, as in the OFDM example, does not work well here: the ISFFT-transformed data isn't constant-modulus like OFDM's per-subcarrier QPSK, so dividing by near-zero values amplifies the noise floor.)
  • Using the same 100 MHz bandwidth and 256-symbol frame length as the OFDM example gives identical range resolution (1.5 m), unambiguous velocity (~2.4 km/s), and velocity resolution, making the two notebooks directly comparable.
  • The key structural difference from OFDM is where the data lives: OFDM assigns data per time-frequency symbol (and typically repeats it symbol-to-symbol for simplicity), while OTFS spreads every DD-domain symbol across the entire frame — this is what gives OTFS its resilience to fast time-varying (high-Doppler) channels in communication systems, and is also why radar processing naturally produces a joint delay-Doppler estimate from one frame rather than two sequential FFT stages.
  • Like OFDM, OTFS can carry a full data payload outside the pilot's guard region — the same waveform used for ranging could simultaneously carry a communication link (joint radar-communication).
