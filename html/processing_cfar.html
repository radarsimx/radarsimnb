CFAR Detection with RadarSimPy¶

Introduction¶

This notebook demonstrates RadarSimPy's CFAR (Constant False Alarm Rate) detection capabilities. RadarSimPy provides built-in implementations of CA-CFAR and OS-CFAR algorithms for adaptive target detection in radar systems.

What You'll Learn:

  • Use RadarSimPy's CFAR functions for 1D detection
  • Configure CA-CFAR (Cell-Averaging CFAR) with cfar_ca_1d()
  • Configure OS-CFAR (Ordered Statistics CFAR) with cfar_os_1d()
  • Compare CFAR performance with different parameters
  • Apply CFAR to simulated radar range profiles

Key RadarSimPy Features Demonstrated:

  • CFAR algorithms: cfar_ca_1d(), cfar_os_1d() for 1D detection
  • Flexible parameters: Guard cells, training cells, false alarm rate
  • Multiple detectors: Square-law and linear detector types

Why This Matters: CFAR enables adaptive thresholding in varying noise environments, improving target detection reliability. RadarSimPy's CFAR functions make it easy to implement and evaluate these algorithms.

CFAR Background¶

CFAR (Constant False Alarm Rate) adjusts detection thresholds based on local noise levels:

  1. Estimates background noise from neighboring cells (training window)
  2. Calculates adaptive threshold to maintain constant false alarm probability
  3. Compares test cell against threshold for detection

RadarSimPy CFAR Algorithms:

  • CA-CFAR (Cell-Averaging): Averages power in training cells
  • OS-CFAR (Ordered Statistics): Uses k-th ordered statistic for robustness

Key Parameters:

  • guard: Number of guard cells on each side of test cell
  • trailing: Number of training cells on each side
  • pfa: Probability of false alarm (e.g., 1e-4)
  • detector: "squarelaw" or "linear"
  • k: Rank for OS-CFAR (typically 3/4 of total training cells)

Create RadarSimPy Transmitter¶

Configure FMCW waveform using RadarSimPy's Transmitter class:

As shown in the diagram below, f and t are used to define the waveform modulation. For a linear frequency-modulated continuous waveform (FMCW), it can be specified as f = [fstart, fend] and t = [tstart, tend]. If t is a single number t = t0, which is equivalent to t = [0, t0]. The bandwidth of the FMCW is abs(fstart - fend). prp is the pulse repetition period, and prp >= (tend - tstart).

|                       prp
|                  +-----------+
|
|            +---f[1]--->  /            /            /
|                         /            /            /
|                        /            /            /
|                       /            /            /
|                      /            /            /     ...
|                     /            /            /
|                    /            /            /
|                   /            /            /
|      +---f[0]--->/            /            /
|
|                  +-------+
|                 t[0]    t[1]

Process with RadarSimPy Functions¶

Compute Range Profile¶

Use RadarSimPy's range_fft() to compute range profile:

from scipy import signal
import radarsimpy.processing as proc

range_window = signal.windows.chebwin(radar.sample_prop["samples_per_pulse"], at=60)
range_profile = proc.range_fft(baseband, range_window)

Apply RadarSimPy CFAR Detection¶

RadarSimPy provides CFAR functions in the processing module:

1D CFAR Functions:

  • cfar_ca_1d(): Cell-Averaging CFAR for 1D data
  • cfar_os_1d(): Ordered Statistics CFAR for 1D data

2D CFAR Functions (for range-Doppler maps):

  • cfar_ca_2d(): Cell-Averaging CFAR for 2D data
  • cfar_os_2d(): Ordered Statistics CFAR for 2D data

This example demonstrates 1D CFAR on range profiles. For 2D CFAR examples, see the FMCW radar notebooks.

In [12]:
ca_cfar = proc.cfar_ca_1d(
    np.abs(range_profile[0, 0, :]) ** 2,
    guard=2,
    trailing=10,
    pfa=1e-4,
    detector="squarelaw",
)
os_cfar = proc.cfar_os_1d(
    np.abs(range_profile[0, 0, :]) ** 2,
    guard=0,
    trailing=10,
    k=14,
    pfa=1e-4,
    detector="squarelaw",
)

Observe CFAR Performance¶

The plot shows:

  1. Range Profile (blue): Two target peaks at different ranges with noise floor
  2. CA-CFAR Threshold (orange): Smooth adaptive threshold following noise level
  3. OS-CFAR Threshold (green): More robust to outliers, slightly higher threshold

Key Observations:

  • Both CFAR algorithms adapt threshold to local noise level
  • OS-CFAR is more robust to interfering targets in training cells
  • Guard cells prevent target energy from biasing threshold estimate
  • Detection occurs where range profile exceeds CFAR threshold

Summary and Key Takeaways¶

RadarSimPy CFAR Functions Used¶

This notebook demonstrated:

  1. cfar_ca_1d() - Cell-Averaging CFAR:

    • Averages power in training cells
    • Simple and computationally efficient
    • Good for homogeneous noise environments

  2. cfar_os_1d() - Ordered Statistics CFAR:

    • Uses k-th ranked value from training cells
    • More robust to interfering targets
    • Better for cluttered environments

  3. Key Parameters:

    • guard: Prevents target leakage into training cells
    • trailing: Number of cells for noise estimation
    • pfa: False alarm probability (1e-4 = 0.01% false alarm rate)
    • detector: "squarelaw" for power, "linear" for amplitude
